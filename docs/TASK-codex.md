## タスク1（最優先）: 文脈FAB（Speed Dial ＋ アイコン差し替え）
**狙い:** 「作る場所＝今いる画面」。タップ＝今いる文脈のものを作る。

### 挙動（タブ別）
| タブ | FABアイコン | タップしたら |
|---|---|---|
| ホーム | ＋ | **Speed Dial** 展開：記録 / つぶやき / 結果 |
| タイムライン | ＋ | 同上の Speed Dial（記録 / つぶやき / 結果） |
| 予定 | カレンダー＋ | 予定作成フォームを**直接**開く |
| お知らせ | ベル＋ | お知らせ作成を**直接**開く |
| ノート（フォルダ内） | ノート＋ | **開いているフォルダに**記事作成を直接開く |

- **長押し機能は無し**（OS干渉リスクのため不採用）。各画面の＋でその画面のものを作る、が主導線。

### アニメ・見た目ルール（合意済み・厳守）
- **タブ切替の合図は「アイコンの差し替え」だけ**。**色は1つのアクセント色で統一（変えない）**。**回転・点滅・消えて再登場はしない**。差し替えは短く上品に（150〜200ms）。
- **Speed Dial開閉のときだけ ＋↔✕ を回転**（ホーム/タイムラインのみ）。直接作成タブの＋は**回さない**（タップ＝即フォーム）。
- **画面遷移・タブ切替時は Speed Dial を必ず閉じて＋へリセット**（前画面のメニューが浮いたまま残る事故を防ぐ）。
- `prefers-reduced-motion` 時はアニメ無しで即切替。

### Speed Dial のガードレール（前回の「縦長・改行だらけ」失敗を防ぐ）
- 各アクション＝**1行**（アイコン＋ラベル横並び）、**`whitespace-nowrap`** で折り返し禁止。
- コンテナ幅は**内容ぴったり（`w-max`/`w-fit`）**。全幅にしない。
- メニューは**絶対配置のオーバーレイ**（Portal / `position:absolute`）でFABの上に浮かせる。**通常フローに入れて周囲を押し広げない**。
- 3項目固定・項目間の余白は小さく。開いたら背景を薄暗く、1タップで閉じる。

**受け入れ条件**
- [x] ホーム/タイムラインの＋で Speed Dial が**3行・内容幅・レイアウトを押し広げない**形で出る（縦長/折り返しにならない）
- [x] 予定/お知らせ/ノートの＋は種別選択を挟まず**直接フォーム**を開く（＋は回らない）
- [x] タブ切替で**アイコンだけ差し替わる**（色は不変・回転/点滅なし）
- [x] ノートのフォルダ内＋で、初期選択が**開いていたフォルダ**（「最上位が選択されるバグ」を解消）
- [x] フォーム/作成を✕で閉じると**開いた画面に戻る**（既存の `router.back()` パターン踏襲）
- [x] 画面遷移時に Speed Dial が閉じ＋へリセットされる
- [x] `prefers-reduced-motion` でアニメ無効化／`aria-label` が状態に追従
- [x] `tsc --noEmit` が通る

## タスク2（重い・再設計）: ノートを「記事リスト形式」に
**狙い:** 「1つの本文を編集し続ける」をやめ、`フォルダ＝ノート` の中に `記事（タイトル＋本文）` を追加していく形式（大会結果リストのイメージ）。

詳細仕様は **`docs/NOTES-PLAN.md` を本タスクに合わせて改訂してから**実装すること（現状の単一本文前提なら書き換える）。

- データ: フォルダ（既存ノート相当）配下に **記事(article)** テーブル/列を追加。記事は `title` ＋ `body` ＋ 作成者・日時。
  - マイグレーションは新タイムスタンプ＋冪等。RLS も忘れず（フォルダの公開範囲・編集権限を継承）。
- **個人ノートも他人が書き込める設定**を可能に（編集権限の対象を「全員/指定/自分」から選べる。共有ノートと同じ仕組みを個人にも拡張）。
- 一覧 = フォルダを開くと記事リスト（タイトル＋日時）。記事タップで詳細。

**受け入れ条件**
- [ ] フォルダ内に記事を複数追加・編集・削除できる
- [ ] 個人ノートでも、設定により他人が記事を追加/編集できる
- [ ] 編集権限が RLS で担保されている（UIガードだけにしない）
- [ ] `NOTES-PLAN.md` が新形式に更新済み
- [ ] マイグレーションは本番DBへ適用し、`tsc`/`build` を通してから push

## タスク3: スプレッドシート取込のプレビューを「編集可能」に
**狙い:** 取込前に間違いをその場で直せる（元シートへの往復をなくす）。

- 取込前プレビュー = **編集可能な表**（先頭〜全行）。各行に**バリデーションのエラー表示**。
- 「○件取込 / △件エラー」のサマリ。**1行の不備で全体を拒否しない**（直す or その行だけスキップ）。
- 列はヘッダー名でマッピング（位置依存にしない）。
- 関連: `src/components/features/ScheduleSheetsManager.tsx`、`src/app/api/google/sheets/create/route.ts`、取込側の処理。

**受け入れ条件**
- [ ] プレビュー上でセルを修正してそのまま取込できる
- [ ] 行ごとにエラー理由が出る（例「3行目: 日付の形式が不正」）
- [ ] 正常行だけ取込→残りは修正して再取込ができる
- [ ] `tsc --noEmit` が通る
