# UI棚卸し（ガラパゴス・使いにくさ点検 2026-06-20）

> 新機能（ノート・フォルダ・CSV一括入力）追加後の統一性/使いやすさ点検。
> 規約は `docs/UI-UNIFICATION.md`。担当: Claude=横断的な小修正 / Codex=ノート・予定まわり。

## 重大（統一性の核）

### 1. 「作成(＋)」の入口がバラバラ ★最優先
現状、同じ「作る」操作なのに経路が機能ごとに違う:
- 練習記録 / つぶやき / 大会結果 … **FABのみ**
- 予定 / お知らせ … **FAB＋画面ヘッダーの＋作成＋マイページ管理メニュー**（3経路）
- 練習メニュー … FAB＋予定カードの「メニューを追加」
- ノート / フォルダ … **FABのみ**（ノート画面に＋が無い）

→ **規約を決めて統一**:
- 「その画面の中身を作る」ものは **画面ヘッダー右の＋**（schedule/notices方式）に統一。
- 横断的なクイック作成は FAB。
- まず **ノート画面(`/notes`)にヘッダー＋を追加**（schedule/noticesと揃える＆発見性向上。今は空状態で「右下の作成メニューから」と案内せざるを得ない）。

### 2. FABが過積載
「投稿する」FABに7項目＋ノートの2段サブメニュー。**投稿(流れる)・ノート(残す)・管理(予定/お知らせ/メニュー)が混在**し、`aria-label="投稿する"` もノート/予定には不適切。
→ いずれか:
- (a) シート内を「投稿 / ノート / 管理」の小見出しで区切る、または
- (b) ノート/フォルダ作成は #1 でノート画面側に寄せ、FABからは外して項目を減らす。

### 3. 用語のゆれ（テーマ/フォルダ/共有）
共有ノートの分類が、NotesViewでは「フォルダ」、詳細のBadgeでは「共有」、当初仕様では「テーマ」と混在。
→ **「フォルダ」に統一**（ラベル・空状態・Badge表記）。

## 中

### 4.（修正済 by Claude / 9ced878）作成フォームを別画面から開くと✕で元に戻らない
マイページ→「予定/お知らせを作成」→✕ で元のマイページに戻らずその画面に留まる問題。`?compose=1`(autoOpen)時は閉じたら `router.back()` で元画面へ戻すよう修正済。
→ 同じ「別画面から?compose的に開く」導線を今後増やす場合は同じ方式に。

### 5. マイページの作成リンクが重複
「予定を作成」「お知らせを作成」は FAB と重複。#4で戻れるようにはなったが、整理（残す/減らす）を検討。

## 小
- マイページの管理メニュー内「その他」(ロール管理/練習場所)が details で1段深い。許容範囲だが、項目が少ないので展開なしの直リンクでもよい。
- ノート個人タブは公開ノートの素朴な一覧。今は可。将来、人別の見せ方は要検討（NOTES-PLAN参照）。
