# UI・データ更新ガイドライン

このアプリで機能を追加・変更するときの共通ルール。

## データ取得

- ページの初期データは Server Component で取得する。
- タブ再訪時は `next.config.ts` のルーターキャッシュ（動的30秒、静的180秒）が使われる。
- 絞り込みだけで済むタブ切替は、取得済みデータをクライアント側で絞り込む。
- 個人情報やログイン状態に依存するSupabaseクエリは、明示的に共有キャッシュしない。
- 大きな一覧は「先頭から件数を増やして再取得」ではなく、将来的にカーソル方式へ移行する。

## データ更新

- いいね、出欠、コメントなどの局所操作は楽観更新または対象行だけ更新する。
- 成功後に一覧全体を再取得しない。
- 失敗時は変更前の状態へ戻し、操作箇所の近くにエラーを表示する。
- `router.refresh()` は、プロフィール変更など複数領域のServer Componentへ影響する場合だけ使う。
- ルーターキャッシュ内の表示とローカルstateが競合しないよう、局所操作は現在のClient Component内で完結させる。
- 作成・編集・削除はSupabase RLSで本人または権限保持者に制限する。UI上の非表示だけを認可に使わない。

## UI

- カードに付随する閲覧情報はカード内へ展開する。
- 本人が所有する項目の操作入口は右上の `…` に統一する。
- 「編集／削除」の選択は共通ボトムシートを使う。
- 短い編集フォームはボトムシートまたはインライン、長いフォームは全画面を使う。
- 削除は共通確認ダイアログを通す。
- 空表示、読込中、保存中、エラーを必ず区別する。
- タップ領域は原則36px以上を確保する。

## コメント

- 吹き出しボタンで投稿カード内へ展開する。
- 初回展開時だけ取得し、以後はローカルstateを維持する。
- 自分のコメントにだけ `…` を表示する。
- 追加・編集・削除は対象行と件数だけを更新する。
