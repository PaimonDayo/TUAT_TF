# UI・操作の統一ガイド（実装指示）

複数人・複数端末で積み増した結果、画面ごとに UI/操作がバラバラ（ガラパゴス化）している。
これを解消するための**確定方針**と**やることリスト**。今後はこの規約に沿って実装する。

> 運用ルール: `origin/master` が正。**force-push 禁止**。作業前に `git pull`。
> マイグレーションは新しいタイムスタンプで、**冪等**（`IF NOT EXISTS` / `DROP POLICY IF EXISTS` 等）に書く。

---

## 1. 確定した統一方針

### A. モーダルは「選ぶ＝シート / 書く＝全画面」で統一
- **ボトムシート（下から）= 軽い“選ぶ”操作だけ**
  - 例: FABの投稿種別選び、`⋯` の編集/削除メニュー、フィルター選択（学年など）、出欠者一覧の表示。
- **全画面モーダル = ちゃんと入力する“書く”フォーム**
  - 例: 練習記録 / つぶやき / 予定 / お知らせ / 大会結果(PB) / 会場 / 目標 / 練習メニュー の入力。
- 理由: フォームをシートに入れると高さ伸縮でガクつく・キーボードで隠れる・窮屈。全画面は高さ固定で安定。
- 既に `ScheduleForm` は全画面化済み（`src/components/ui/fullscreen.tsx` を使用）。これに他フォームも寄せる。

### B. 編集・削除は「⋯ ActionMenu」に統一（現行標準・2026-06-21 現状反映）
- すべてのコンテンツ（投稿・つぶやき・コメント・予定・大会結果・会場・お知らせ・ロール・ノート記事）で、
  右上などの **`⋯`（`ActionMenu`）→「編集」「削除」** に統一する。これが**現在の実装標準**。
- 「編集」で該当の**全画面フォーム**が開く。「削除」は **`ConfirmDialog` で確認**してから実行。
- 破壊的操作には必ず確認を挟む（誤操作防止）。

> 【未採用の将来案】「左スワイプ＋長押し＋Undoスナックバー」案を `docs/UX-ISSUES-2026-06.md` #4 に記載しているが、
> **現状は未実装**。共通基盤（Undo）が無いため、当面は上の `ActionMenu` を正とする。
> 採用する場合は全画面横断で一括置換し、中途半端に混在させないこと。

### C. 進め方
1. まず**共通部品**と本ガイド（規約）を整える。
2. 以後は**必ず共通部品で作る**。
3. 既存画面は**主要画面から順に**共通部品へ寄せる（いきなり全部はやらない）。

## 2. 共通部品（`src/components/ui` に集約）

### 既存（流用）
`Button` / `Card` / `Input` / `Textarea` / `Badge` / `Sheet`(=選ぶ) / `FullScreen`+`FullScreenContent`(=書く) / `SegmentedControl` / `Toggle`

### 追加・標準化する部品
- **`FormModal`**: 全画面フォームの共通ガワ。`FullScreenContent` をベースに、
  ヘッダー（×＋タイトル）／スクロール本文／（必要なら）下部固定の保存ボタン。**全フォームはこれを使う**。
- **`ActionMenu`**: `⋯` を押すと出るシート。`編集` / `削除`（破壊的は赤＋確認）を共通化。
  props 例: `onEdit?`, `onDelete?`, `editLabel`, `deleteLabel`。
- **`Disclosure`**: `▾` で開閉する共通の折りたたみ（予定のアクセス、グラフ詳細などで使う）。
- **`KeyValue`**: 「ラベル：値」の項目表示（結果・補強・感想・参加費など）。本文は `Linkify` 経由＋`whitespace-pre-wrap`。
- **`EmptyState`** / **`Skeleton`**: 空状態・読み込み中の共通表現。
- 入力系の見た目（余白・角丸・高さ）は `Input`/`Textarea`/`Card` の基準に必ず合わせる。
- 日付/時刻入力は `globals.css` で iOS のはみ出し対策済み。`type="date/time"` をそのまま使う。

## 4.5 システム/アーキテクチャの統一（UIだけでなく仕組みも揃える）

画面ごとに取得・更新・キャッシュのやり方がバラついているので、以下を標準とする。

### データ取得
- 画面の**初期データは Server Component + `src/lib/queries.ts` の関数**で取得する。画面に直接 supabase クエリを書かない。
- **新しいクエリは必ず `lib/queries.ts` に集約**（重複・流儀の分散を防ぐ）。
- クライアントでの追加取得（操作起点のもの：会場リスト、コメント、出欠など）はブラウザ用クライアントで。だが定義は使い回せる形に。

### キャッシュ / 表示速度（“重いページの標準形”）
- `next.config.ts` の `staleTimes`（dynamic 30 / static 180）を**維持**。
- 主要な遷移リンク（下タブ、管理メニュー等）は **`prefetch`** を付ける。
- 重いセクションは **`Suspense` でストリーミング**し、ガワ（プロフィール等）を即表示。`(app)/loading.tsx` のスケルトンも活用。
- → 「重い一覧を待ってからページ全体を出す」をやめ、**軽い所を先に出して重い所を流し込む**を全画面で標準化。

### 更新（ミューテーション）
- **楽観的更新（即UI反映）＋必要なときだけ `router.refresh()`**。
- **全ページ再取得（router.refresh）の多用は禁止**。SessionKeepAlive がトークン更新ごとに refresh して全体がもたついた事例あり（→ SIGNED_OUT のみ refresh に修正済）。
- 件数表示などの軽い反映はローカル state で。破壊的操作は確認を挟む。

### 認証 / セッション
- 検証・更新は **proxy(`getUser`)**、ページは **`getCurrentProfile`(`getSession`、ネット往復なし)**。
- `SessionKeepAlive` は自動更新（startAutoRefresh）は維持しつつ、**TOKEN_REFRESHED では refresh しない**（SIGNED_OUT のみ）。
- データのアクセス制御は最終的に **RLS** が担保（UIで緩めても安全）。

### 権限
- サーバー側の出し分けは **`permissionsOf(profile.roles)`**、DB側は **`can_*()` 関数で RLS**。
- **UIガードとRLSの両方**で必ず守る（片方だけにしない）。

### その他の仕組み
- 新バージョン検知（`VersionWatcher` → 更新バナー）を全体で維持。
- Realtime は現状未使用。必要になったら別途方針を決める。
- エラー/空/読み込みの表現は `EmptyState` / `Skeleton` に共通化（UIセクションと対）。

## 6. タイムラインの絞り込み・簡易表示（2026-06-20 追加）

### 学年フィルター（実装済 / `GradeMenu` / `TimelineView`）
- **問題だった点**: 旧 `GradeFilter` は選択中の学年名をボタンに表示するため幅が変わりガクつく＆単一選択。
- **採用案＝iOS風プルダウンメニュー**（`src/components/features/GradeMenu.tsx`）:
  - 「学年 ▾」ボタン→押すと**チェック式のドロップダウン**が出て**複数学年を選択**。選んだ学年にチェックマーク。
  - ボタンは選択数のみ表示（「学年 2」）で**幅がほぼ変わらない**。`value: string[]`（空＝全学年）。
  - ブロック(`SegmentedControl`)・フォロー/簡易(丸トグル)と同じ行に並べ、トーン統一。
- タイムライン・メンバー一覧の両方を `GradeMenu` に統一済み。旧 `GradeFilter`(シート単一選択) は廃止。

### 簡易表示（コンパクト表示）の追加
- タイムラインの1件1件が大きいので、**詳細表示／簡易表示を切り替えるトグル**を用意する。
- 簡易表示では `RecordCard` / `TweetCard` を1〜2行程度の要約（名前・種別・要点・日時）に圧縮。タップで詳細（または展開）。
  - 実装案: 各カードに `compact?: boolean` を渡し、compact 時は本文を数行省略・補強/感想等の詳細やアクション列を畳む。
- 切替トグルは絞り込みバーの一部として置く（`SegmentedControl`「詳細/簡易」か小さなアイコントグル）。選択状態はローカルstateで保持。

## 7. 確定ルール追補（2026-06-21）— 詳細は `docs/UX-ISSUES-2026-06.md`

### ガクつき禁止（全コンポーネント共通の鉄則）
選択や数値で要素の幅/高さが変わるのは禁止（安っぽい）。出欠・チップ・トグル・カウンタ全部に適用。
1. 数字は `font-variant-numeric: tabular-nums`。
2. 最長文言ぶんを `min-width` で確保（「出席/欠席」等の切替で伸縮させない）。
3. 0でも要素を消さない（`visibility:hidden` で場所だけ残す。`display:none` 禁止）。
4. 状態差は**色だけ**で表す（幅を変えない）。
5. `contain: layout` で波及を隔離。
6. 読み込み中はスケルトン。

### フィルタの「適用中」表示＝固定幅チップ
- 適用中の学年等は外せる**固定幅チップ**（`B1` `M1`…）。上のガクつき禁止に準拠（tabular-nums＋min-width）。
- **学年フィルタは在籍者がいる学年だけ動的表示**（0人は出さない）。

### メンバー一覧・ロール管理は学年でグループ分け
- B1/B2/…/M1/M2/D1… の見出しで区切る。名前検索も追加。

### 練習記録フォームはブロック別（2系統）
- 中長距離＝現行 / 短距離・跳躍・投擲＝共通（メニュー・目的/意識・タイム・感想）。
- 出し分けの初期値は**プロフィールの専門種目（所属ブロック）**。プロフィール編集に専門種目設定を追加。

### スプレッドシート取込のプレビューは編集可能に
- 取込前プレビュー＝**その場で直せる表**（行単位エラー表示→修正→そのまま取込）。元シートへの往復をなくす。

### 横断
- 二重送信防止（保存中は無効化＋「保存中…」）。無言update対策を全更新へ横展開。
- 破壊的操作の確認は `ConfirmDialog` に統一。タップ領域は最低44px。
