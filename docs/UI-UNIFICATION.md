# UI・操作の統一ガイド（実装指示）

複数人・複数端末で積み増した結果、画面ごとに UI/操作がバラバラ（ガラパゴス化）している。
これを解消するための**確定方針**と**やることリスト**。今後はこの規約に沿って実装する。

> 運用ルール: `origin/master` が正。**force-push 禁止**。作業前に `git pull`。
> マイグレーションは新しいタイムスタンプで、**冪等**（`IF NOT EXISTS` / `DROP POLICY IF EXISTS` 等）に書く。

---

## 1. 確定した統一方針

### A. モーダルは「選ぶ＝シート / 書く＝全画面」で統一
- **ボトムシート（下から）= 軽い“選ぶ”操作だけ**
  - 例: FABの投稿種別選び、`⋯` の編集/削除メニュー、フィルター選択（学年など）、出欠者一覧の表示。
- **全画面モーダル = ちゃんと入力する“書く”フォーム**
  - 例: 練習記録 / つぶやき / 予定 / お知らせ / 大会結果(PB) / 会場 / 目標 / 練習メニュー の入力。
- 理由: フォームをシートに入れると高さ伸縮でガクつく・キーボードで隠れる・窮屈。全画面は高さ固定で安定。
- 既に `ScheduleForm` は全画面化済み（`src/components/ui/fullscreen.tsx` を使用）。これに他フォームも寄せる。

### B. 編集・削除は「⋯」メニューに統一
- すべてのコンテンツ（投稿・つぶやき・コメント・予定・大会結果・会場・お知らせ）で、
  **右上などの `⋯`（MoreHorizontal）→ シートで「編集」「削除」** に統一。
- コメントの**長押し方式は廃止**し、`⋯` に統合（操作の発見性とPC対応のため）。
- 「編集」を押すと該当の**全画面フォーム**が開く。「削除」は確認後に削除。

### C. 進め方
1. まず**共通部品**と本ガイド（規約）を整える。
2. 以後は**必ず共通部品で作る**。
3. 既存画面は**主要画面から順に**共通部品へ寄せる（いきなり全部はやらない）。

## 2. 共通部品（`src/components/ui` に集約）

### 既存（流用）
`Button` / `Card` / `Input` / `Textarea` / `Badge` / `Sheet`(=選ぶ) / `FullScreen`+`FullScreenContent`(=書く) / `SegmentedControl` / `Toggle`

### 追加・標準化する部品
- **`FormModal`**: 全画面フォームの共通ガワ。`FullScreenContent` をベースに、
  ヘッダー（×＋タイトル）／スクロール本文／（必要なら）下部固定の保存ボタン。**全フォームはこれを使う**。
- **`ActionMenu`**: `⋯` を押すと出るシート。`編集` / `削除`（破壊的は赤＋確認）を共通化。
  props 例: `onEdit?`, `onDelete?`, `editLabel`, `deleteLabel`。
- **`Disclosure`**: `▾` で開閉する共通の折りたたみ（予定のアクセス、グラフ詳細などで使う）。
- **`KeyValue`**: 「ラベル：値」の項目表示（結果・補強・感想・参加費など）。本文は `Linkify` 経由＋`whitespace-pre-wrap`。
- **`EmptyState`** / **`Skeleton`**: 空状態・読み込み中の共通表現。
- 入力系の見た目（余白・角丸・高さ）は `Input`/`Textarea`/`Card` の基準に必ず合わせる。
- 日付/時刻入力は `globals.css` で iOS のはみ出し対策済み。`type="date/time"` をそのまま使う。

## 5. 用語メモ
- 「モーダル」= 入力時に上に出るパネル（シート or 全画面）。
- 「シート」= 下からせり上がる小窓（`Sheet`）。
- 「全画面」= 画面全体を覆う入力画面（`FullScreen`）。
