# 文言ガイドライン（部員向けの言葉づかい統一）

最終更新: 2026-07-23 / Claude

> 目的: アプリ内の日本語文言を、非エンジニアの部員が迷わず読める言葉に統一する。
> 「同期・反映・取込・書き戻し・正・上書き」などの**開発都合の用語が画面に出ている**問題を解消するための規約。
> UI・操作の規約本体は `docs/UI-UNIFICATION.md`、UX方針は `docs/UX-ISSUES-2026-06.md`。本書は**言葉づかいだけ**を扱う。

---

## 0. 大原則

1. **部員は「アプリ・スプレッドシート・Google」しか知らない**前提で書く。DB・同期方向・列・行・トークンなどの内部概念は画面に出さない。
2. **敬体（です・ます）で統一**。常体（〜だ／〜する）や体言止めの説明文は使わない。
3. **質問形（「どちらに記録する？」等）は使わない**。アプリの文言としてくだけすぎる。ラベルは名詞、操作は動詞、案内は「〜します／〜してください」。
4. **短く。1文1情報**。1つの吹き出し・注釈に情報を詰め込まない。
5. **不安をあおる語を避ける**。「上書き」「失敗」「エラー」「消えます」は、必要なときだけ・必ず対処法とセットで。

---

## 1. 文体の型（どこに何を書くか）

| 要素 | 文体 | 例 |
|---|---|---|
| 入力欄のラベル | **体言止め（名詞）** | 「日付」「場所」「メニュー」 |
| ボタン | **動詞・言い切り** | 「記録する」「保存する」「予定に登録」 |
| 補助説明（欄の下の注釈） | **敬体・1文** | 「選ぶと、練習記録が1時間ごとに自動で同期されます」 |
| プレースホルダ | **「例: 〜」または「〜を入力」** | 「例: 100m 11.2 (+1.5)」「場所名を入力」 |
| 成功メッセージ | **敬体・完了を伝える** | 「保存しました」「すべて既読にしました」 |
| エラー・失敗 | **§3の型**（何が起きた＋どうすれば） | 「保存できませんでした。もう一度お試しください」 |
| 確認ダイアログ | タイトル＝結果／本文＝敬体の補足 | 「ログアウトしますか？」「次に使うときは再度ログインが必要になります」 |

---

## 2. 用語の言い換え表（開発用語 → 部員向け）

**画面に出す文言では、左の語を使わず右へ置き換える。** コメント・変数名・DB列名・ログは対象外（そのままでよい）。

| 開発用語（禁止） | 部員向け（推奨） | 補足 |
|---|---|---|
| 反映（する／される） | **書き込む／表示される／登録する** | 「どこに」何が起きるかで動詞を選ぶ。スプシへ=書き込む、アプリ画面に=表示される、予定として=登録する |
| 同期 | **自動で書き込み／自動で取り込み**、または「連携」 | ボタン名など短くしたい所は「連携」可。「同期」単独は残さない |
| 取込・取り込み | **取り込み**（ひらがな送り）／「アプリに読み込む」 | |
| 書き戻し | **スプレッドシートに書き込む** | |
| 正（になる／が正） | 使わない。**動作で説明**する | 「アプリで記録すると、スプレッドシートにも自動で書き込まれます」 |
| 上書き（されます） | 原則出さない。必要時のみ「**新しい内容に置き換わります**」 | 「上書き」は不安語。挙動を淡々と説明する |
| 反映に失敗 | **書き込めませんでした** | |
| 発行（する） | **作る／作成する** | 「シートを発行」→「シートを作る」 |
| プルダウン | **選択欄／メニュー** | 「対象ブロックのプルダウンを自動設定」→「対象ブロックの選択欄をあらかじめ用意」 |
| 正常行／エラー行 | **問題のない行／エラーのある行** | |
| 未出力 | **未書き込み** | 管理画面のラベル |
| 競合スキップ | **重複のため見送り** | |
| 入力元 | **記録の保存先** | 「記録の入力元」→「記録の保存先」 |
| リコンサイル／切替前の同期 | 使わない。**「設定の変更」**でまとめる | |
| write-through / LWW 等の英語 | 画面に出さない | コード内コメントのみ |

> 迷ったら: **「主語をシステムにしない。部員の操作か、アプリ/スプシの動作で書く」**。
> 例）×「スプシが正になります」→ ○「アプリで記録すると、スプレッドシートにも自動で書き込まれます」。

## 3. エラー・失敗メッセージの型

**必ず「①何が起きたか ②どうすればいいか」の2部構成。**原因が内部的なものは部員に見せない。

- ○「保存できませんでした。もう一度お試しください」
- ○「スプレッドシートに『補強』の欄がなかったので、その項目だけ書き込めませんでした」
- ×「スプレッドシートへの反映に失敗しました（自動で再試行されます）」← 「反映」「再試行」が固い
- ×「HTTP 500」「同期の進捗情報がありません」← 内部エラーは「うまくいきませんでした。時間をおいてお試しください」に丸める

**部員本人に非がないエラー（同期の連続失敗など）は、対処できる運営者にだけ出す。**一般部員に「3回連続で失敗しています」とだけ見せない。

---

## 4. 運営者向け画面の扱い

運営者（権限者）だけが見る同期・シート発行・システム状態の画面も、**同じ用語表に従う**。「運営者は分かるだろう」で開発用語を残さない（運営者も多くは非エンジニア）。
ただし、集計値（取込 X / 更新 Y など）は「新しく取り込み X件 / 更新 Y件」のように**平易語＋件数**で残してよい。
